# Forming - AI 기반 초등 학습 도구

## 비전

1명의 개발자가 AI와 함께, 초등학생이 수학/과학을 **직접 손으로 풀고 AI 튜터에게 피드백 받는** 학습 도구를 만든다.

핵심 철학: **AI Agentic** — 코드는 데이터 흐름과 UI만 담당하고, 판단과 생성은 전부 AI에게 맡긴다.

## 초기 구상

### 해결하려는 문제
- 초등학생에게 맞는 **적응형 학습**이 부족함
- 기존 학습 앱은 객관식 클릭 위주 → 풀이 과정을 보지 않음
- 선생님/학부모가 일일이 채점하고 설명하는 데 한계

### 접근 방식
- **캔버스에 직접 풀이를 쓰게 하고**, AI가 Vision으로 분석
- 교육과정 데이터를 기반으로 **문제를 자동 생성**
- 틀린 문제의 약점을 분석해서 **다음 문제를 추천**
- 하드코딩 대신 **프롬프트로 모든 판단을 AI에게 위임**

### 타겟 사용자
- 초등학교 5~6학년 학생 (1차)
- 학부모 (향후 리포트 기능)
- 선생님 (향후 학급 관리)

## 기술 스택

```
Frontend:  Next.js 16 (App Router) + React 19 + Tailwind CSS 4
Canvas:    @excalidraw/excalidraw 0.18
Math:      KaTeX + remark-math + rehype-katex
AI:        Claude Code CLI (claude -p, stream-json for vision)
Model:     claude-sonnet-4-6
Data:      JSON (교육과정, 문제) + JSONL (학생 이력)
Locale:    AI agentic (프롬프트 기반, 번역 테이블 없음)
```

## 핵심 아키텍처

### 학습 플로우
```
홈(교육과정 선택) → 단원 페이지(문제 목록) → 문제 풀이(캔버스)
    → AI 분석(Vision) → 결과/피드백 → 추천 문제 or 튜터 채팅
```

### AI 호출 구조
```
프롬프트 템플릿 (.md)  +  교육과정 컨텍스트 (JSON)  +  로케일 (tutorPrompt)
    → prompt-loader.ts ({{변수}} 치환)
    → claude.ts (CLI 호출)
    → JSON 파싱 → 결과 반환
```

### 난이도 프레임워크 (직교 개념축)
```
난이도 = 축 개수 × 연쇄 깊이 × 함정 수

Level 1 (기초): 1축, 연쇄 없음, 함정 없음  → 80-95% 정답률
Level 2 (응용): 2축, 1단 연쇄, 0-1 함정   → 50-80% 정답률
Level 3 (심화): 3-4축, 2-3단 연쇄, 1-2 함정 → 15-50% 정답률
```

### 데이터 구조
```
src/data/
├── curriculum/          # 교육과정 (커밋됨)
│   ├── grade5-math.json
│   ├── grade6-math.json
│   └── grade6-science.json
├── problems/            # 정적 문제 (커밋됨)
├── prompts/             # AI 프롬프트 템플릿 (커밋됨)
├── generated/           # AI 생성 문제 (gitignored)
└── history/             # 학생 이력 JSONL + 캔버스 이미지 (gitignored)
```
