# Claude Design Brief — Forming

> **사용법**: claude.ai → Claude Design 세션을 시작할 때 이 문서 전체를 첫 메시지로 붙여넣으세요. 이후 화면별 세부 프롬프트는 아래 "화면별 요청"을 참고.

---

## 1. Product

**이름**: Forming
**한 줄 설명**: 한국 초등학생~고등학생(3–12학년)을 위한 AI 기반 수학/과학 학습 도구
**메타 설명**: "AI와 함께하는 수학/과학 학습 도구"

**핵심 기능**:
- 학년별 교육과정에 맞춘 문제 풀이 (초3–고3)
- 손글씨(perfect-freehand) 캔버스에 풀이 과정 기입
- AI가 풀이를 분석하여 약점 파악 + 개인화 피드백
- AI 튜터가 개념을 대화형으로 설명
- 난이도 4단계: 기초/보통/심화/도전

**타깃 사용자**:
- **1차**: 한국 초등학생 (특히 3–6학년) — 부모 감독 하에 사용
- **2차**: 중/고등학생 자율 학습
- **교사/학부모**: 향후 리포트 기능 고려 중

**경쟁/레퍼런스**:
- 콴다 (한국, 문제 풀이 앱)
- 밀크북, 아이스크림홈런 (한국 초등 디지털 학습)
- Khan Academy Kids (미국, 초등 UX 참고)
- Duolingo (게이미피케이션, 학습 습관화)

---

## 2. Tech Stack (디자인 제약 조건)

- **Framework**: Next.js 16 (App Router)
- **Styling**: Tailwind CSS 4 (CSS-first config via `@theme inline` in `src/app/globals.css`)
- **Font**: 현재 Geist Sans + Geist Mono (변경 가능 — 초등학생에게 더 적합한 폰트 제안 환영)
- **한글 폰트 필수**: Pretendard, Noto Sans KR, Spoqa Han Sans Neo 등 한국어 가독성 우선
- **Canvas**: perfect-freehand (SVG 기반 손글씨)
- **수식 렌더링**: KaTeX
- **다크 모드**: 현재 없음 (light only) — 야간 학습 고려해 추가 여부 판단 필요
- **반응형**: 현재 `max-w-4xl` 위주, 모바일 최적화 약함 → 태블릿 우선 (초등학생은 태블릿 많이 사용)

**현재 디자인 토큰** (`src/app/globals.css`):
```css
--background: #f9fafb  /* gray-50 */
--foreground: #111827  /* gray-900 */
--font-sans: Geist
--font-mono: Geist Mono
```

**현재 사용 중인 색**: 파란색(primary), 초록(성공), 빨강(오답), 보라(심화), 주황(도전), 노랑(난이도 별).

## 3. 디자인 방향 요청

### 필수 요건
1. **한국어 UI 100%** — 모든 버튼/헤딩/마이크로카피는 한국어. 영어 혼용 금지.
2. **초등학생 친화적** — 친근하고 학습 의욕을 돋우지만 유치하지 않음. "똑똑해 보이는 재미"가 목표.
3. **접근성** — 터치 타깃 ≥ 48px, 본문 16px 이상, 대비 WCAG AA 이상.
4. **태블릿 우선 반응형** — 초등학생은 태블릿을 많이 씀. 모바일/데스크톱도 지원.
5. **손글씨 캔버스 친화적** — 풀이 공간이 넓고 깔끔해야 함. 버튼이 캔버스를 방해하면 안 됨.

### 원하는 성격 (모두 OR — 하나만 강하게 가도 됨)
- **친근한 프리미엄**: 차분하고 신뢰감 있지만 따뜻한 (Duolingo보다 조용함, Linear보다 친근함)
- **교육적 놀이감**: 게임화 요소 살짝, 그러나 공부가 본질
- **손글씨 튜터**: 종이·연필 감성 + 디지털 정교함 (Apple Pencil/굿노트 ethos)

### 피하고 싶은 것
- 유치한 캐릭터나 만화풍 일러스트 과용
- 보라색 그라디언트 CTA (AI 슬롭 패턴)
- 좌우 대칭 3컬럼 피처 그리드
- Inter/Roboto/Helvetica 같은 중립 폰트 (한국어 가독성 떨어짐)
- 하얀 배경에 파란 버튼만 있는 "Khan Academy 클론" 느낌 (현재 상태 — 탈피 목표)

## 4. 화면별 요청 (프로토타입 생성 대상)

### Screen 1 — 홈/커리큘럼 선택 (`/`)
- 학년 → 학기 → 단원 트리 탐색
- 최근 학습 이어하기 / 추천 문제
- 브랜드 아이덴티티가 처음 드러나는 곳

### Screen 2 — 단원/난이도 선택 (`/chapter/[chapterId]`)
- 단원 내 소주제 리스트
- 난이도 4단계 (기초/보통/심화/도전) 선택
- 학습 진도율 시각화

### Screen 3 — 문제 풀이 (`/problem/[problemId]`)
- 상단: 문제 텍스트 + 다이어그램(SVG)
- 중앙: **큰 손글씨 캔버스** (핵심)
- 도구: 지우개/되돌리기/초기화/제출
- 계산 공간이 부족하지 않아야 함

### Screen 4 — 결과/피드백 (`/result/[submissionId]`)
- 정/오답 즉시 피드백 + 감정 배려 (오답 시 좌절 방지)
- AI 분석: 어디서 막혔는지, 무엇을 다시 볼지
- 다음 추천 문제 / 튜터로 넘어가기

### Screen 5 — AI 튜터 (`/tutor/[chapterId]`)
- 대화형 레슨 (설명 → 확인 질문 → 재설명 반복)
- 이미지/수식/다이어그램 혼합
- "이해했어요!" / "다시 설명해주세요" 같은 반응 버튼

### Screen 6 — 개념 설명 (`/theory/[topicId]`)
- 교과서처럼 읽기 좋은 장문 (Markdown + KaTeX)
- 다이어그램과 예제 통합

## 5. 산출물 요청

1. **비주얼 방향 2–3개 변형** (프로토타입) — 위 6개 화면 중 3–4개를 대표 화면으로
2. **디자인 토큰** — 색상(hex), 타이포(font-family, size/weight scale), 간격(spacing scale), 라운딩(border-radius), 그림자, 모션(duration/easing)
3. **컴포넌트 스펙** — 버튼(primary/secondary/ghost/destructive), 카드, 입력, 알림(success/error/info), 진행 표시(StepIndicator)
4. **한국어 폰트 결정** — Pretendard/Noto Sans KR/Spoqa 중 추천 + 이유
5. **다크 모드 여부** — 권장 여부 + 이유 (야간 학습 고려)
6. **Claude Code Handoff 번들** — 선택된 방향을 Tailwind 4 `@theme inline` 형식의 CSS 변수 + 컴포넌트 React/TSX 스니펫으로
